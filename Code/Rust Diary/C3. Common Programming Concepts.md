# 3.1 Variables and Mutability 
- By default, variables are immutable.
```rust 
fn main(){
	let x = 5;
	println!("x = {x}");
	x = 6;
	println!("x = {x}");
}
```
output will be
```bash
$ cargo run
   Compiling variables v0.1.0 (file:///projects/variables)
error[E0384]: cannot assign twice to immutable variable `x`
 --> src/main.rs:4:5
  |
2 |     let x = 5;
  |         - first assignment to `x`
3 |     println!("The value of x is: {x}");
4 |     x = 6;
  |     ^^^^^ cannot assign twice to immutable variable
  |
help: consider making this binding mutable
  |
2 |     let mut x = 5;
  |         +++

For more information about this error, try `rustc --explain E0384`.
error: could not compile `variables` (bin "variables") due to 1 previous error

```
we will get compile-time errors when we attempt to change a immutable value
This help you to mistakenly change a value that you design to not change in the first part. This type of error can be difficult to track down, especially when the second piece of code change the value only sometimes.
But you also make variable to mutable by adding `mut`
```rust
fn main(){
	let mut x = 5;
	println!("x = {x}");
	x = 6;
	println!("x = {x}");
}
```
the output will be 
```bash
$ cargo run
   Compiling variables v0.1.0 (file:///projects/variables)
    Finished `dev` profile [unoptimized + debuginfo] target(s) in 0.30s
     Running `target/debug/variables`
x = 5
x = 6

```
## Declaring Constants
Constants are also a values that are bound to a name and **are not allowed** to change
- Difference from immutable value
 1. You **ARE NOT** allowed to use `mut` with constants. Constants are **always** immutable
 2. Declare constants using the `const` instead of `let` and the type of the value must be annotated. [[See more in "Data Type"]]
 Example of a constant declaration
 ```rust
 const THREE_HOURS_IN_SECONDS: u32 = 60 * 60* 3;
 ```
 **Note:**  naming convention for constants is to use all uppercase with underscores between words. 
## Shadowing
The second variable is what the compiler see
```rust
fn main() {
    let x = 5;

    let x = x + 1;

    {
        let x = x * 2;
        println!("The value of x in the inner scope is: {x}");
    }

    println!("The value of x is: {x}");
}
```
 output 
 ```bash
 $ cargo run
   Compiling variables v0.1.0 (file:///projects/variables)
    Finished `dev` profile [unoptimized + debuginfo] target(s) in 0.31s
     Running `target/debug/variables`
The value of x in the inner scope is: 12
The value of x is: 6

 ```
 shadowing is different from marking a variable as `mut`
 - We'll get a compile-time error if we reassign to this variable without using `let`
 - Using `let` we can change a value but have the variable be immutable after those transformations have completed.
 - `let` can also change the type of the value but reuse the same name
```rust
let spaces = "   ";
let space = spaces.len();
```
- First `spaces` variable is a string type
- Second `spaces` variable is a number type
Saving time to come up with different name `spaces_num, spaces_str`
**However**
```rust
let mut spaces = "  ";
spaces = spaces.len();
```
The error says we're not allowed to mutate a variable's type

## Scalar Types
A scalar type represents a single value. Rust has four primary scalar types: `integers, floating-point number, Booleans, and characters`. 
***1. Integer Types

| Length                     | Signed  | Unsigned |
| :------------------------- | :------ | :------- |
| **8-bit**                  | `i8`    | `u8`     |
| **16-bit**                 | `i16`   | `u16`    |
| **32-bit**                 | `i32`   | `u32`    |
| **64-bit**                 | `i64`   | `u64`    |
| **128-bit**                | `i128`  | `u128`   |
| **Architecture-dependent** | `isize` | `usize`  |
**Note:** Each signed variant can store number from -(2^(n-1)) to 2^(n-1) - 1
=> i8 can store -2^7 to 2^7 -1 = -128 to 127
`isize` and `usize` types depend on the architecture of the computer whether its 64 bits or 32 bits
You can write integer literals in any form of table below 

| Number literals    | Example       |     |
| :----------------- | :------------ | --- |
| **Decimal**        | `98_222`      |     |
| **Hex**            | `0xff`        |     |
| **Octalt**         | `0o77`        |     |
| **Binary**         | `0b1111_0000` |     |
| **Byte (u8 only)** | `b'A'`        |     |
### Integer Overflow
- When overflow occur if you're compiling in debug mode your program will `panic` at runtime. [[See more in chapter 9]]
- if you're compiling with `--release flag`, and overflow occur rust performs' two complement wrapping.

## Floating-Point Types
Rust's floating-point types are `f32` and `f64`
- `f32`: 32 bits
- `f64`: 64 bits
- `f64` is default type
```rust
fn main(){
	let x = 2.0; //f64
	
	let y: f32 = 3.0;
}
```
Floating-point numbers are represented according to the IEEE-754 standard
## Numeric Operations
```rust
fn main() {
    // addition
    let sum = 5 + 10;

    // subtraction
    let difference = 95.5 - 4.3;

    // multiplication
    let product = 4 * 30;

    // division
    let quotient = 56.7 / 32.2;
    let truncated = -5 / 3; // Results in -1

    // remainder
    let remainder = 43 % 5;
}
```
## The Boolean Type
- Two possible value `true` and `false`
- Size: 1 byte (8 bits)
```rust
fn main() {
    let t = true;

    let f: bool = false; // with explicit type annotation
}
```
## The Character Type
```rust
fn main() {
    let c = 'z';
    let z: char = 'ℤ'; // with explicit type annotation
    let heart_eyed_cat = '😻';
}
```
- `Char` use single quotation `' '`, while string literals use double quotation  `" "`
- Size: 4 bytes (32 bits)
- Represent a Unicode scalar value, which means it can represent a lot more than just ASCII. Accented letters; Vietnamese, Chinese, Japanese, and Korean characters; emojis; and zero spaces are all valid `char` values in Rust. [[More detail in chapter 8]]
## Compound Types
- Can group multiple values into one type
- Two primitive compound types: `tuples` and `arrays`
### The Tuple Type
- General way of grouping together a number of a values with a variety of types into one compound type.
- Fixed length: once declared, cannot grow or shrink in size.
```rust
fn main(){
	let tup: (i32, i64, u8) = (500, 6.4, 1);
}
```
- Tuple is considered a single compound element. Therefore, to get the individual values out of a tuple
```rust
fn main(){
	let tup: (i32, i64, u8) = (500, 6.4, 1);

	let (x, y, z) = tup;
	
	println!("The value of y is: {y}");
}
```
- We can also access a tuple element directly by using a period (.) followed by the index of the value we want to access.
```rust
fn main() {
    let x: (i32, f64, u8) = (500, 6.4, 1);

    let five_hundred = x.0;

    let six_point_four = x.1;

    let one = x.2;
}
```
Tuple without any value has a special name, `unit`. This value and its corresponding type are both written `()` and represent an empty value or an empty return type. Expressions implicitly return the unit value if they don’t return any other value.
